CSS Transitions

Why transitions?

  • There's a basic common set of interactions designers want to be able to control without writing Javascript
  • These are simple but core interactions like changing colors and size when a mouse is hovering over something, to having elements fade in and out (as opposed to a jaring single frame movement)
  • Instead of having property changes take effect immediately, you can cause the changes in a property to take place over a period of time.
  • This is part of the idea of putting as much functionality into CSS as possible.
  • Remember, load as much into your HTML as you can, then pass the buck to your CSS, and finally use JS as a last resort (a highly common last resort)
  • Big Picture: There are transitions, transforms and animations - All three combine help make content come ALIVE! In ways that visual styling and display/position/float can't
  • Your pages are now dynamic, MOVING things

What properties can be transitioned

Only certain properites can be transitioned, and the idea is simple:

A property can be transitioned if it has a definable 'middle' point

This is why a left: 0 -> left: 100px can be transitioned, we know it's middle point of 50px;

But this is why we can't transition position: relative -> position: static, what does the middle point look like?

Another thing that will animate inconsistently is the "auto" property, so going from margin: 10px 10px -> margin: 10px auto might be buggy in some browsers

Transitionable properties:

  • background-color
  • background-position
  • border-color
  • border-width
  • border-spacing
  • bottom
  • clip
  • color
  • crop
  • font-size
  • font-weight
  • height
    • left
  • letter-spacing
  • and so much more... line-height; margin; max-height; max-width; min-height; min-width; opacity; outline-color; outline-offset; outline-width; padding; right; text-indent; text-shadow; top; vertical-align; visibility; width; word-spacing;

When transitions end

I want to plant a seed now for when we start talking about Javascript...

How can we hook into when a transition ends so we can do more stuff with Javascript?

We often code up the concept of "events" in our browser, the idea is that we write code for when certain events happen

:hover, :active, :focus can be thought of as three events, and via CSS we tie behavior to them (changing properties)

But we want a much bigger list, so keep in mind that when transitions finish running, the browser emits an event called "transitioned"

We'll write pseudo-code as such:

 
              function tellMeTransitionIsDone() {
                when: #coolDiv.is.transitioned
                do: alert('div is done!),
                do: getDivContents
                do: sendDivContentsToServer
              }

This is just pseudo-code, it's not properly written JS, it will break if you try and run it, but it conveys the idea.
